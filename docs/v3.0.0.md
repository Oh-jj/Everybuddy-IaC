# v3.0.0 — WAF 업로드 허용 확장 + Bastion → RDS 접근 허용

**날짜:** 2026-05-17 ~ 2026-05-18  
**커밋:** `5cf7d06` · `d8bc902` · `666f31a`

---

## 배경

### WAF False Positive 추가 수정

v2.10.0에서 도입된 `AllowBinaryUploadEndpoints` 룰에 누락된 엔드포인트 2개를 추가.  
운영 중 Sampled Requests 분석을 통해 `CommonRuleSet`이 정상 요청을 차단하는 false positive 확인.

### Bastion → RDS 직접 접근 허용

로컬 개발 환경에서 RDS 데이터를 직접 조회할 필요가 생김.  
SSH 터널(Local Port Forwarding) 방식으로 Bastion을 경유해 RDS에 안전하게 접근하기 위해  
RDS Security Group에 Bastion SG 출처 인바운드 규칙 추가.

---

## ✏️ Changed

### modules/waf/main.tf — AllowBinaryUploadEndpoints 룰 확장

```
추가된 엔드포인트:
- /api/v1/users/me  : 프로필 이미지 업로드 (JPEG/PNG 바이너리 → false positive)
- /api/v1/messages  : 채팅 파일 전송 (이미지, 비디오, 오디오, 문서)
```

### main.tf — RDS allowed_sg_ids 확장

```hcl
# Before
allowed_sg_ids = {
  backend         = module.security.backend_sg_id
  private_backend = module.security.private_backend_sg_id
}

# After
allowed_sg_ids = {
  backend         = module.security.backend_sg_id
  private_backend = module.security.private_backend_sg_id
  bastion         = module.security.bastion_sg_id
}
```
