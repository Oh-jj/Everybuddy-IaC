# v2.8.0 — 외부 GPU 서버 → Loki 로그 수집 허용

**날짜:** 2026-05-09
**커밋:** `dev` — GPU 서버 Loki 푸시 보안그룹 규칙 추가

---

## 배경

GPU 서버(laurel)에서 실행 중인 Promtail이 모니터링 서버의 Loki로 로그를 푸시하려 했으나, 보안그룹 규칙 누락으로 TCP 연결 자체가 차단되어 `context deadline exceeded` 오류가 반복되던 상황.

기존 `monitoring_loki` 규칙은 `source_security_group_id = aws_security_group.backend.id` 로만 허용되어 AWS VPC 외부 IP는 전혀 차단.

---

## 🆕 Added

### modules/security — `monitoring_loki_from_gpu` 규칙

외부 GPU 서버의 CIDR에서 Loki 포트(3100)로의 인그레스를 허용하는 규칙 추가.

```hcl
resource "aws_security_group_rule" "monitoring_loki_from_gpu" {
  count             = length(var.gpu_server_cidrs) > 0 ? 1 : 0
  type              = "ingress"
  description       = "Loki push endpoint from external GPU server"
  from_port         = 3100
  to_port           = 3100
  protocol          = "tcp"
  cidr_blocks       = var.gpu_server_cidrs
  security_group_id = aws_security_group.monitoring.id
}
```

`count` 조건으로 `gpu_server_cidrs`가 비어 있으면 규칙 자체가 생성되지 않음.

### modules/security/variables.tf — `gpu_server_cidrs` 변수

```hcl
variable "gpu_server_cidrs" {
  description = "External GPU server CIDRs allowed to push logs to Loki (port 3100)"
  type        = list(string)
  default     = []
}
```

### variables.tf (root) — `gpu_server_cidrs` 변수

```hcl
variable "gpu_server_cidrs" {
  description = "External GPU server CIDRs allowed to push logs to Loki (port 3100)"
  type        = list(string)
  default     = ["x.x.x.x/32"]
}
```

## 아키텍처 변화

```
외부 GPU 서버 (laurel)
       │
       │  TCP :3100  (신규 SG 규칙으로 허용)
       ▼
  EC2 everybuddy-monitoring
  └── Loki :3100  ← Promtail 로그 수신
```

기존에는 Loki로의 인그레스가 AWS 내부 SG 참조(`backend` SG)만 허용되어 있었기 때문에, VPC 외부에 위치한 GPU 서버의 트래픽은 보안그룹 레벨에서 차단됨.

## 운영 참고

- GPU 서버 IP 변경 시 `gpu_server_cidrs` 값만 수정 후 `terraform apply`
- Loki가 `0.0.0.0:3100`으로 리슨 중인지 모니터링 서버에서 주기적 확인 권장 (`ss -tlnp | grep 3100`)
