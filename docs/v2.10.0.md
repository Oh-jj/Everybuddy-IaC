# v2.10.0 — 봇 차단 강화 + S3 VPC Endpoint (NAT GW 비용 절감)

**날짜:** 2026-05-16
**커밋:** `dev` — WAF Rate-Based Rule · Anonymous IP List 추가 / S3 VPC Gateway Endpoint 추가

---

## 배경

개발 단계에서 외부 봇 스캐닝 트래픽이 지속 유입되어 ALB LCU·WAF 요청 과금이 발생.
기존 WAF는 알려진 악성 IP(IP Reputation List)만 차단하여 미등록 봇 IP는 통과.
아울러 Private Backend → S3 파일 트래픽이 NAT GW를 경유해 데이터 처리 비용($0.059/GB) 이 발생하고 있었음.

---

## 🆕 Added

### modules/waf — Rate-Based Rule (priority 0)

단일 IP가 5분간 200건을 초과하면 자동 Block. 모든 룰 중 최우선(priority 0) 평가.

```hcl
rule {
  name     = "RateLimitPerIP"
  priority = 0

  action { block {} }

  statement {
    rate_based_statement {
      limit              = 200
      aggregate_key_type = "IP"
    }
  }
}
```

**200건/5분 선정 근거:**
- 개발자 4-5명 환경: 초당 0.7건 여유 → 정상 테스트에 충분
- 봇 패턴(현재 로그): 동일 초에 2건씩 연속 → 수십 초 내 임계 도달 후 차단

### modules/waf — Anonymous IP List (priority 5)

VPN 출구 노드, 익명 프록시, Tor 경유 트래픽 차단.

```hcl
rule {
  name     = "AWSManagedRulesAnonymousIpList"
  priority = 5

  override_action { none {} }

  statement {
    managed_rule_group_statement {
      name        = "AWSManagedRulesAnonymousIpList"
      vendor_name = "AWS"
    }
  }
}
```

> **운영 전환 시 재검토 필요:** 글로벌 서비스 출시 후 VPN 사용 유저 차단 여부를 정책적으로 결정할 것.

### modules/networking — S3 VPC Gateway Endpoint

Private Backend → S3 트래픽을 AWS 내부망으로 직접 연결. NAT GW 데이터 처리 비용 제거.

```hcl
data "aws_region" "current" {}

resource "aws_vpc_endpoint" "s3" {
  vpc_id            = aws_vpc.main.id
  service_name      = "com.amazonaws.${data.aws_region.current.name}.s3"
  vpc_endpoint_type = "Gateway"
  route_table_ids   = [aws_route_table.private.id]
}
```

- **비용:** 무료 (Gateway Endpoint는 AWS 무과금)
- **코드 변경:** 없음 (라우팅 레벨 변경, 앱 투명)
- **폴백:** 엔드포인트 장애 시 NAT GW로 자동 우회

## WAF 룰 우선순위 전체 정리

| Priority | 룰 이름 | 동작 | 역할 |
|----------|---------|------|------|
| 0 | `RateLimitPerIP` | Block | 200건/5분 초과 IP 즉시 차단 |
| 1 | `AWSManagedRulesAmazonIpReputationList` | Block | 알려진 봇·스캐너·악성 IP |
| 2 | `AllowTranslateEndpoints` | Allow | /speech, /text 명시 허용 |
| 3 | `AWSManagedRulesCommonRuleSet` | Block | OWASP Top 10 |
| 4 | `AWSManagedRulesKnownBadInputsRuleSet` | Block | PHP, Log4j 등 |
| 5 | `AWSManagedRulesAnonymousIpList` | Block | VPN, 프록시, Tor |
