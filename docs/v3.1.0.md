## 배경

### AWS Bedrock 모니터링 에이전트 도입

외부 GPU 서버(laurel)의 이상 상태를 자동으로 감지하고 분석하기 위해  
AWS Lambda + EventBridge + Bedrock(Claude) 기반 모니터링 에이전트를 구축.  
Prometheus 메트릭을 주기적으로 수집하여 이상 징후 발생 시 Bedrock이 원인을 분석하고 Slack으로 알림 전송.

> 에이전트 상세 설계, Lambda 로직, 알림 포맷은 `docs/bedrock_agent/` 참고.

### WAF 영상 업로드 허용

백엔드팀 요청으로 `/api/v1/translate/video` 엔드포인트 추가 허용.  
영상 번역 기능 도입에 따라 최대 50MB 영상 파일 업로드가 WAF SizeRestrictions 룰에 차단되는 문제 해결.

## ✨ Added

### modules/bedrock_agent/ (신규 모듈)

| 리소스 | 설명 |
|--------|------|
| `aws_lambda_function` | GPU 모니터링 에이전트 (Python 3.12) |
| `aws_cloudwatch_event_rule` | EventBridge 주기 트리거 |
| `aws_cloudwatch_event_target` | Lambda 연결 |
| `aws_cloudwatch_log_group` | Lambda 실행 로그 (14일 보관) |
| `aws_ssm_parameter` | Slack Webhook URL (SecureString 암호화) |
| `aws_lambda_permission` | EventBridge → Lambda 호출 권한 |

### modules/security/main.tf — Lambda 전용 Security Group 추가

```
lambda_gpu_monitor_sg:
  Ingress: 없음 (Lambda는 트리거 기반, 인바운드 불필요)
  Egress:  전체 허용 (VPC 내 Prometheus 조회 + NAT GW 경유 Bedrock/Slack 호출)
```

### variables.tf — Bedrock 에이전트 변수 추가

| 변수 | 용도 |
|------|------|
| `slack_webhook_url` | GPU 알림 Slack Incoming Webhook |
| `existing_role_arn` | Lambda 실행 IAM Role ARN |

## 최종 WAF AllowBinaryUploadEndpoints 엔드포인트 목록

| 엔드포인트 | 허용 사유 |
|------------|-----------|
| `/api/v1/translate/speech` | m4a 음성 파일 업로드 |
| `/api/v1/translate/text` | 대용량 텍스트 번역 요청 |

| `/api/v1/messages` | 채팅 파일 전송 |

## Bedrock 에이전트 동작 흐름

```
EventBridge (주기 트리거)
  └── Lambda (VPC 내 실행)
        ├── Prometheus API 조회 (GPU 메트릭 수집)
        ├── 이상 감지 시 → Bedrock Claude 호출 (원인 분석)
        └── 분석 결과 → Slack 알림 전송 (NAT GW 경유)
```

> 상세 구현 내용은 `docs/bedrock_agent/overview.md` 참고.
