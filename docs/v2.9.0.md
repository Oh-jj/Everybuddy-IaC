# v2.9.0 — 인스턴스 업그레이드 + WAF 번역 엔드포인트 허용 규칙

**날짜:** 2026-05-13
**커밋:** `dev` — Bastion/Private Backend 스펙 상향, WAF AllowTranslateEndpoints 규칙 추가

---

## 배경

### 인스턴스 업그레이드

서비스 안정성 확보를 위해 Bastion 및 Private Backend EC2 인스턴스 타입을 상향.
Bastion은 CI/CD ProxyJump 트래픽 처리 안정성, Private Backend는 Spring Boot + 번역 요청 증가에 따른 메모리/CPU 여유분 확보가 목적.

### WAF 번역 엔드포인트 허용

v2.7.0에서 추가된 WAF CommonRuleSet이 번역 API 요청을 차단하는 문제 발생.
- `/api/v1/translate/speech`: m4a 음성 파일 업로드 → WAF 기본 Body 사이즈 검사(8KB) 초과
- `/api/v1/translate/text`: 긴 텍스트 번역 시 Body 크기 초과 가능성

번역 엔드포인트를 CommonRuleSet보다 높은 우선순위로 명시적 허용 처리.

### modules/waf/main.tf — AllowTranslateEndpoints 규칙 추가 및 우선순위 재배치

번역 API 엔드포인트를 CommonRuleSet 이전(priority 2)에 명시적으로 허용하는 규칙 삽입.
기존 Rule 2(CommonRuleSet) → priority 3, Rule 3(KnownBadInputs) → priority 4로 이동.

```hcl
rule {
  name     = "AllowTranslateEndpoints"
  priority = 2

  action { allow {} }

  statement {
    or_statement {
      statement {
        byte_match_statement {
          search_string         = "/api/v1/translate/speech"
          positional_constraint = "STARTS_WITH"
          field_to_match { uri_path {} }
          text_transformation { priority = 0; type = "NONE" }
        }
      }
      statement {
        byte_match_statement {
          search_string         = "/api/v1/translate/text"
          positional_constraint = "STARTS_WITH"
          field_to_match { uri_path {} }
          text_transformation { priority = 0; type = "NONE" }
        }
      }
    }
  }
  ...
}
```

## 최종 WAF 규칙 우선순위

| Priority | Rule | 동작 |
|----------|------|------|
| 1 | AWSManagedRulesAmazonIpReputationList | Block (악성 IP) |
| 2 | AllowTranslateEndpoints | **Allow** (번역 API 명시 허용) |
| 3 | AWSManagedRulesCommonRuleSet | Block (OWASP Top 10) |
| 4 | AWSManagedRulesKnownBadInputsRuleSet | Block (PHP, Log4j 등) |

> priority 숫자가 낮을수록 먼저 평가됨. 번역 엔드포인트는 priority 2에서 Allow 처리되므로 priority 3 CommonRuleSet 검사 대상에서 제외됨.
